# 导入提示词进入工作台精调方案

## 背景
当前系统主流程是：上传原始 Markdown -> 大模型生成大纲与逐页 prompt -> 工作台精调 -> 批量生图 -> 导出 PPT。

新增需求是支持“用户已经有每一页提示词”的场景，让用户可以直接导入提示词进入系统，并继续在工作台中精调，然后批量生图并下载 PPT。

导入来源需要支持两种形式：

- 一个 ZIP 文件
- 多个 `.md` 文件

## 目标
新增一条面向“已有逐页提示词”的业务链路：

导入提示词 -> 自动结构补全 / 大纲提取 -> 进入工作台精调 -> 批量生图 -> 导出 PPT

其中：

- 用户原始 prompt 是核心资产，系统不得在导入和结构补全过程中自动改写 prompt 正文
- 导入后必须有进度条，明确展示“结构补全 / 大纲提取”处理进度
- 导入项目需要进入现有工作台体系，而不是单独做一个轻量预览页

## 范围
本次只设计文档，不实施。

本次方案覆盖：

- 导入入口
- 文件解析规则
- 导入后项目建模
- 自动结构补全 / 大纲提取任务
- 工作台适配
- 进度条与状态说明
- 批量生图与 PPT 导出衔接

本次不包含：

- JSON 导入
- 导入后自动重写 prompt
- 新建独立编辑器或第二套工作台
- 实际代码实现细节

## 用户流程

### 1. 入口
在项目管理页提供两个入口：

- 新建素材项目
- 导入提示词

在新建页增加模式切换：

- 上传原始 Markdown
- 导入已有提示词

### 2. 导入文件
导入模式支持：

- 上传单个 ZIP
- 一次上传多个 `.md`

规则：

- ZIP 和多个 `.md` 不能混传
- 只识别 `.md`
- 忽略 `index.md`
- 忽略隐藏文件、目录、`__MACOSX`、非 `.md` 文件

### 3. 创建项目
导入成功后立即创建项目，并把每个 `.md` 文件映射为一页 slide：

- `slide.prompt` 保存文件全文
- `slide.title` 初始取 Markdown 首标题，没有则取文件名
- 其它结构字段先填最小默认值
- 直接初始化默认 `style_guide`
- 项目标记为导入型项目

### 4. 自动结构补全 / 大纲提取
项目创建后立即启动后台任务，对导入 prompt 集合做结构化处理：

- 逐页提取标题、页面类型、核心表达、版式建议等
- 汇总整套提示词的大纲信息
- 保存结构化结果
- 不改写原始 prompt 正文

### 5. 跳转工作台
导入成功后直接跳到工作台页。

如果结构补全任务还在执行：

- 工作台顶部显示进度条
- 用户可以先看页面列表和 prompt
- 工作台对写操作（保存 prompt、新增/删除页、一致性检查、修正不一致）为只读，避免与后台 worker 的整包写发生 lost update
- 结构字段区域可显示"正在补全"
- 任务通常 1 分钟内完成，完成后自动解锁全部精调能力

### 6. 后续操作
结构补全完成后，用户可以：

- 继续逐页修改 prompt
- 新增页 / 删除页
- 检查一致性
- 修正不一致
- 进入批量生图
- 下载 PPT

## 项目模型设计

### 项目来源
新增 `project_origin` 字段：

- `generated_markdown`
- `imported_prompts`

### 导入项目状态
建议新增状态：

- `prompts_imported`
- `import_structure_generating`
- `import_structure_generated`

### 导入项目的最小初始数据
导入创建时至少具备：

- `project_id`
- `project_origin=imported_prompts`
- `generation_options`
- `style_guide`
- `slides[]`
- `generation_state=prompts_imported`

其中每页 slide 初始字段包括：

- `slide_no`
- `id`
- `title`
- `prompt`
- `page_type=""`
- `page_role=""`
- `core_message=""`
- `layout=""`
- `modules=[]`
- `visual_elements=[]`
- `text_hierarchy=""`
- `page_text=[]`

## 文件解析规则

### 排序
页面顺序按文件名排序生成：

1. 文件名前缀数字优先
2. 无数字时按文件名字典序

例如：

- `01-cover.md`
- `02-overview.md`
- `10-summary.md`

会按 1, 2, 10 排序。

### 标题提取
每页初始标题优先级：

1. Markdown 第一条标题
2. 第一条非空文本
3. 文件名去扩展名

### prompt 保存
每个文件全文直接保存到 `slide.prompt`，不做清洗改写。

## 自动结构补全 / 大纲提取

### 目标
导入项目需要进入工作台精调，因此仅有 `prompt` 不够，还需要补出结构化信息供现有工作台使用。

### 输出内容
逐页补全：

- `title`
- `page_type`
- `page_role`
- `core_message`
- `layout`
- `modules`
- `visual_elements`
- `text_hierarchy`
- `page_text`

项目级补全：

- `deck_brief`
- 可选 `slide_count_plan`

### 约束
该任务只能做“提取、归纳、结构化”，不能改写：

- `slide.prompt`

### 触发方式
导入完成后自动触发后台任务，无需用户手动点击。

## 进度条设计

### 总体原则
“结构补全 / 大纲提取”必须有明确进度条和阶段说明，避免用户不知道系统在做什么。

复用现有任务模型和 `JobProgress` 组件，不新做第二套进度系统。

### 任务类型
新增任务类型：

- `import_structure_generation`

### 阶段定义
建议新增阶段：

- `queued`：任务已创建
- `import_scanning`：扫描导入文件
- `import_parsing`：解析逐页提示词
- `import_outline_extracting`：提取页面结构
- `import_brief_generating`：汇总整体大纲
- `import_structure_saving`：保存结构化结果
- `completed`：完成
- `failed`：失败

### 阶段文案
进度条展示的用户文案建议如下：

- 扫描导入文件
- 解析逐页提示词
- 提取页面结构
- 汇总整套提示词的大纲信息
- 保存结构化结果

### message 示例
运行中 message 应尽量具体，例如：

- 正在解析第 3/12 页提示词
- 正在提取页面结构（8/12）
- 正在汇总整套提示词的大纲信息
- 正在保存结构化结果

### progress 分配
建议进度分段：

- 文件扫描 / 建页：0.00 - 0.15
- 逐页结构提取：0.15 - 0.80
- 项目级大纲汇总：0.80 - 0.92
- 保存与收尾：0.92 - 1.00

要求：

- 长任务中进度持续推进
- 不要长时间停留在 0%
- 刷新页面后仍可自动接续显示当前任务进度

### 工作台展示
导入项目进入工作台后：

- 顶部显示 `JobProgress`
- 同时展示一行辅助说明：

“正在根据导入提示词补全页面结构与大纲信息，完成后可使用完整精调能力。”

如果任务失败：

- 进度条显示错误信息
- 用户仍可继续手动编辑 prompt

## 工作台适配

### 保留能力
导入项目在工作台中保留：

- 逐页切换
- prompt 编辑与保存
- 新增页
- 删除页
- 一致性检查
- 修正不一致
- 进入批量生图

### 隐藏能力
导入项目不保留这些“重写 prompt”能力：

- 继续生成
- 重生成当前页 prompt
- 重生成全部 prompt

### 大纲按钮语义调整
对导入项目，原“重新生成大纲”按钮不能再表示“重写大纲并联动重写 prompt”。

应改成更准确的语义，例如：

- 重新解析页面结构
- 重新生成结构化大纲

该操作只重新提取结构化信息，不改写 prompt。

## 批量生图与导出

### 生图
导入项目直接复用现有批量生图页：

- 使用当前 `slide.prompt` 生图
- 支持失败页重试
- 支持单页补充要求后重生

### 导出 PPT
生图完成后直接复用现有导出 PPT 接口，无需新增流程。

### 导出闭环
继续保留现有 `prompt_zip` 导出格式，使链路形成闭环：

导出 prompt ZIP -> 外部修改 -> 再导入 -> 工作台精调 -> 生图 -> 导出 PPT

## 验收标准

### 功能验收
- 可以导入 ZIP
- 可以导入多个 `.md`
- 导入后立即进入工作台
- 工作台顶部能看到结构补全 / 大纲提取进度条
- 进度条有阶段说明和动态 message
- 结构补全不会改写原始 prompt
- 导入项目支持逐页 prompt 精调
- 导入项目支持一致性检查、批量生图、PPT 下载

### 体验验收
- 用户知道系统正在处理什么
- 刷新页面后任务进度不丢
- 任务失败时有明确错误提示
- 即使结构任务失败，也不阻塞用户继续手改 prompt

## 风险与注意事项
- 导入 prompt 的质量差异很大，结构提取结果可能不稳定，因此必须保留人工精调能力。
- 导入项目没有原始 Markdown 语料，不应复用“继续生成”这类原始链路语义。
- 工作台需要按 `project_origin` 做能力分流，否则会出现错误按钮或错误预期。
- 进度条若只显示“处理中”而无阶段和 message，会让用户误以为系统卡住。

## 默认决策
- 只支持 ZIP 或多个 `.md`
- 不支持 JSON 导入
- 导入后直接进入工作台
- 自动触发结构补全 / 大纲提取
- 必须显示进度条和阶段说明
- 不自动改写用户 prompt
- 导入项目允许手动精调和后续生图导出
